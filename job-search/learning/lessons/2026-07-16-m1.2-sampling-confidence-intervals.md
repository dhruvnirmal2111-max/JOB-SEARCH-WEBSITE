# Sampling, Standard Error & Confidence Intervals — 2026-07-16
> Module M1 · Statistics & Probability · topic 1.2 · confidence before: 🔴 · builds on the CLT lesson (`2026-07-16.md`)

## Concept

Last lesson: the CLT says **sample means are approximately normal**, with spread **SE = σ / √n**. Today we use that to answer the real question: *"I measured something on a sample. How sure am I about the true value for the whole population?"*

You never see the whole population. You take a **sample**, compute a **statistic** (a mean, a conversion rate), and want to say something about the true value. The sample statistic is your **point estimate**, and it is almost certainly a little off. A **confidence interval (CI)** puts a range around it that quantifies that uncertainty.

```
   Population (unknown truth)
         │   take a random sample of size n
         ▼
   Sample  ── compute ──▶  point estimate  (x̄ for a mean, p̂ for a rate)
         │
         ▼
   SE = σ / √n            how much that estimate would wobble
         │                across many repeated samples
         ▼
   95% CI = estimate ± 1.96 · SE     the range of plausible truth
```

**Building a 95% CI for a mean:**

```
CI  =  point estimate  ±  (critical value × standard error)
    =  x̄  ±  (1.96 × σ/√n)
```

- `x̄` — sample mean (the point estimate)
- `1.96` — the **critical value**: on a normal curve, 95% of the area sits within ±1.96 SE of the centre (rule of thumb ≈ 2)
- `SE = σ/√n` — the standard error from last lesson

The `1.96 × SE` piece is the **margin of error**. It shrinks as `n` grows: bigger sample, tighter interval, more precise estimate.

```
          Sampling distribution of the mean  (≈ normal by the CLT)
                              ▲ density
                             ╱▮╲
                            ╱ ▮ ╲
                           ╱  ▮  ╲
                          ╱   ▮   ╲
                     ────╱    ▮    ╲────
                    ╱         ▮         ╲
        ───────────┼─────────┼─────────┼───────────▶  values of x̄
              x̄ − 1.96·SE    x̄     x̄ + 1.96·SE
                    └──────── 95% ────────┘
              margin of error = 1.96·SE on each side
```

**z vs t (one nuance).** You rarely know the true population σ, so you estimate it from the sample (`s`). For small samples that extra uncertainty matters, so you use the **t-distribution** (slightly fatter tails → slightly wider interval) instead of the normal/z. For large n (≈30+), t and z are nearly identical, so 1.96 is fine. Interviewers like hearing you know *why* t exists: it accounts for estimating σ from the data.

**The interpretation that trips everyone up.** A 95% CI does **not** mean "there is a 95% probability the true value is inside this particular interval." In frequentist stats the true value is fixed; it is either in your interval or it is not. The correct reading:

> *If I repeated this sampling process many times, about 95% of the intervals I construct would contain the true value.*

The confidence is in the **procedure**, not in any single interval.

```
  20 repeated 95% CIs (each ┣━━●━━┫ is one sample's interval; │ = true value)
                              │
     ┣━━●━━┫                  │
        ┣━━●━━┫               │
              ┣━━●━━┫         │
                   ┣━━●━━┫    │
                       ┣━━●━━┫│
                          ┣━━●┿┫
                            ┣━┿●━━┫
                              │┣━━●━━┫
        ┣━━●━━┫  ✗ (misses)   │
                              │
   ≈ 19 of 20 intervals cover the truth; ~1 misses. That is what "95%" means.
```

## Worked example

You run a conversion test on Puffy's checkout. Out of **n = 1,600** visitors, **128** convert → sample rate **p̂ = 0.08 (8%)**.

For a proportion, `SE = √(p̂(1 − p̂)/n)`:

- SE = √(0.08 × 0.92 / 1600) = √(0.0736 / 1600) = √0.000046 = **0.00679** (≈ 0.68%)
- 95% CI = 0.08 ± 1.96 × 0.00679 = 0.08 ± 0.0133 → **[6.7%, 9.3%]**

Report it as: *"Conversion is 8%, 95% CI [6.7%, 9.3%]."* You can be reasonably confident the true rate is in that band, but not that it is exactly 8%. If a stakeholder claims "the new design hits 9%," you cannot rule that out: 9% sits inside the interval, so on this data it is **not distinguishable** from 8%. That single observation is the seed of hypothesis testing (next lesson).

**Sample-size lever.** Want the margin of error halved (±0.66% instead of ±1.33%)? SE scales with 1/√n, so you need **4× the visitors** (~6,400). This is exactly the maths behind "how long must we run the A/B test."

## Interview angle

- **The classic trap:** *"What does a 95% confidence interval mean?"* → give the procedure-based answer. "95% chance the true value is in this range" is the single most common wrong answer, and interviewers listen for it.
- **"How do you get a more precise estimate?"** → increase n (narrows SE), or reduce measurement variance. Quadruple n to halve the margin.
- **CI vs point estimate:** always prefer reporting the interval. A bare "8%" hides the wiggle; showing the CI signals statistical maturity.
- **z vs t:** t is for estimated σ / small samples; both converge for large n.

## Self-check
1. You measure average order value on **n = 100** orders: mean = **$120**, sample SD = **$40**. Give the approximate 95% confidence interval, and say in one line what it tells you.
2. A colleague says: *"Our 95% CI for retention is [30%, 34%], so there's a 95% chance true retention is between 30% and 34%."* What is wrong with that sentence, and how would you say it correctly?
3. Your CI is too wide for the business to act on. Name **two** things you could change to tighten it, and for the sample-size one, roughly how much more data to halve the margin of error.

## Answers
1. **SE = 40/√100 = 40/10 = 4.** 95% CI = 120 ± 1.96×4 = 120 ± 7.84 → **≈ [$112, $128]**. It says: given this sample, the true average order value is plausibly between about $112 and $128; our best single guess is $120 but there is real uncertainty around it.
2. The true retention rate is **fixed**, not random, so a single interval either contains it or does not, there is no "95% probability" for this one interval. Correct version: *"The method that produced [30%, 34%] captures the true retention rate about 95% of the time across repeated samples,"* or loosely, *"we are 95% confident the true rate is in that range,"* where the confidence is in the procedure.
3. (a) **Collect more data** — SE = σ/√n, so to halve the margin of error you need **≈ 4× the sample size**. (b) **Reduce variance** in the measurement — e.g. segment to a more homogeneous group, remove noise/outliers, or use a lower-variance metric. (Accepting a lower confidence level, say 90%, also narrows it, but by trading away certainty rather than adding information.)
