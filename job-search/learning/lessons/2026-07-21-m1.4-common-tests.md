## Concept

Last lesson gave you the *machinery* of hypothesis testing (H0/H1, p-value, α). Today is about **picking the right test**. They're all hypothesis tests — the choice comes down to **what kind of variable you're comparing** and **how many groups**.

```
What are you comparing?
│
├─ MEANS of a numeric outcome (revenue, session length, order value)
│    ├─ 1 group vs a known/target value ....... one-sample t-test
│    ├─ 2 independent groups (A vs B) ......... two-sample (independent) t-test
│    ├─ same units, two conditions (before/after) paired t-test
│    └─ 3+ groups ............................. ANOVA  → then post-hoc (Tukey)
│
└─ CATEGORICAL variables (counts / proportions: converted?, device, channel)
     ├─ association between 2 categoricals ..... chi-square test of independence
     └─ observed vs expected distribution ...... chi-square goodness-of-fit
```

**t-test — compare means (2 groups max).**
- *One-sample:* is this group's mean different from a target (e.g. is avg order value ≠ $100)?
- *Two-sample (independent):* do two separate groups differ (control vs variant AOV)?
- *Paired:* same subjects measured twice (spend before vs after a feature). Paired uses each subject as its own control, so it has **more power** when it applies.
- *z-test vs t-test:* use **z** when the population σ is known or n is large; **t** when you estimate σ from a smaller sample (t has fatter tails). Last lesson's proportion test was a z-test.

**ANOVA (Analysis of Variance) — compare means across 3+ groups.** It answers "do *any* of the groups differ?" with a single test. If significant, it does **not** tell you *which* pair differs — you follow up with a **post-hoc test** (e.g. Tukey HSD).

**Chi-square — for categorical data.**
- *Test of independence:* are two categorical variables associated (is conversion related to device)?
- *Goodness-of-fit:* does an observed distribution match an expected one?
It works by comparing **observed counts** to the **counts you'd expect if there were no relationship**.

**Assumptions (know these):** t-test / ANOVA assume roughly normal data (or large n, via the CLT), independent observations, and similar variances. When those break badly, reach for **non-parametric** versions: **Mann-Whitney U** (↔ two-sample t-test), **Kruskal-Wallis** (↔ ANOVA).

## Worked example

Three questions from a Puffy experiment, three different tests:

1. **"Does average order value differ between control and variant?"** → two numeric means, 2 groups → **two-sample t-test.**
2. **"Do three checkout designs (A/B/C) differ in average session length?"** → 3 groups, numeric mean → **ANOVA**, then Tukey to find which design differs.
3. **"Is converting (yes/no) associated with device (mobile/desktop)?"** → two categoricals → **chi-square test of independence.**

Chi-square, concretely. Observed conversions by device:

```
            Converted    Not      Total
Mobile          40       960      1000
Desktop         70       630       700
            ─────────────────────────────
Total          110      1590      1700
```

Overall conversion = 110 / 1700 ≈ **6.5%**. *If device didn't matter,* you'd expect each cell at that rate:
- Expected mobile conversions = 1000 × 0.065 = **65** (observed **40** — well below)
- Expected desktop conversions = 700 × 0.065 = **45.5** (observed **70** — well above)

Chi-square adds up how far observed sits from expected:

```
χ² = Σ (O − E)² / E     over all cells
```

Just the two conversion cells already contribute (40−65)²/65 + (70−45.5)²/45.5 ≈ 9.6 + 13.2 = 22.8, and the "not converted" cells add more. That's a large χ² → tiny p → **conversion and device ARE associated** (desktop converts noticeably better). The intuition to carry: *big gaps between observed and "expected if independent" = evidence of a real relationship.*

## Interview angle

- **"Which test would you use?"** is one of the most common screening questions. Memorise the decision tree: **t-test = 2 means, ANOVA = 3+ means, chi-square = categorical associations.**
- **"Why not just run t-tests on every pair of 3+ groups?"** → the **multiple-comparisons problem** from M1.3: each test has ~5% Type I error, so many tests inflate the false-positive rate. ANOVA is one omnibus test that controls it; post-hoc tests then correct for the pairwise comparisons.
- **Paired vs unpaired:** use paired when the same units are measured twice — it removes between-subject variance and gives more power.
- **A/B testing mapping:** conversion rate (a proportion) → two-proportion z-test or chi-square; revenue / AOV / time (continuous) → t-test.
- **Assumption failures:** know the non-parametric fallbacks (Mann-Whitney, Kruskal-Wallis) so you're not stuck when data is skewed or tiny.

## Self-check
1. You're comparing average revenue per user across **4 pricing tiers**. Which test, and what do you do next if it's significant?
2. You want to know whether **subscription status (yes/no)** is associated with **acquisition channel (organic / paid / referral)**. Which test?
3. Why is running 6 pairwise t-tests across 4 groups a bad idea, and what's the fix?

## Answers
1. **ANOVA** (means across 3+ groups). If significant, run a **post-hoc test (e.g. Tukey HSD)** to find *which* tiers differ — ANOVA only tells you at least one does.
2. **Chi-square test of independence** — both variables are categorical.
3. Each test carries ~5% Type I error; running 6 pushes the chance of *at least one* false positive well above 5% (multiple comparisons). Fix: **one ANOVA**, then post-hoc comparisons with a correction (Tukey / Bonferroni).
