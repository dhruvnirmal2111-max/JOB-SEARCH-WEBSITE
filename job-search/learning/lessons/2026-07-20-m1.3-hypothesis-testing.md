## Concept

Hypothesis testing is the framework for deciding whether an effect you see in data is **real** or could just be **noise**. It is the formal answer to "is 8% actually different from 9%, or did we get lucky?"

You start with two competing claims:

- **Null hypothesis (H0):** the boring default — *no effect, no difference* (the new checkout converts the same as the old one).
- **Alternative hypothesis (H1):** there *is* an effect (the new checkout is different).

The move that trips people up: you **assume H0 is true**, then ask *how surprising is my data under that assumption?* That surprise is the **p-value**.

> **p-value = the probability of seeing a result at least as extreme as yours, IF the null hypothesis were true.**

Small p-value = your data would be very unlikely under "no effect" = evidence *against* H0.

You compare p to a threshold you set in advance, the **significance level α** (usually **0.05**):

- **p < α** -> reject H0, call it "statistically significant."
- **p ≥ α** -> fail to reject H0 (not "H0 is true," just "not enough evidence").

```
      Null distribution (the world where H0 is true)
                     ▲ density
                    ╱▮╲
                   ╱ ▮ ╲
                  ╱  ▮  ╲
             ────╱   ▮   ╲──░░░░░   <- shaded tail area = p-value
            ╱         ▮      ╲░░░░░     (how far out your result landed)
        ────┼─────────┼───────┼──────▶  test statistic
                   H0 mean   observed
   Far out in the tail -> tiny p -> "this would be a fluke under H0" -> reject H0.
```

Because you can be wrong in two directions, there are **two kinds of error**:

```
                         REALITY
                  H0 true          H0 false (real effect)
              ┌───────────────┬────────────────────────┐
  Reject H0   │  Type I error │       Correct ✓        │
  ("found it")│  false positive│      power = 1 − β      │
              │      = α       │                        │
              ├───────────────┼────────────────────────┤
  Keep H0     │   Correct ✓   │      Type II error     │
  ("nothing") │    = 1 − α    │   false negative = β   │
              └───────────────┴────────────────────────┘
```

- **Type I error (α):** you cry wolf — declare an effect that isn't real. You *control* this directly by choosing α (0.05 = willing to be wrong this way 5% of the time).
- **Type II error (β):** you miss a real effect.
- **Power = 1 − β:** the probability you *detect* a real effect when it exists. Power goes **up** with a bigger sample size, a bigger true effect, or a higher α. Underpowered tests (too little data) miss real effects and produce misleading "no difference" results.

There is a built-in tension: **lowering α** (fewer false positives) **raises β** (more misses, lower power). You can't shrink both at once without more data.

## Worked example

Back to Puffy. Control checkout converts **128 / 1,600 = 8.0%**; the new variant converts **160 / 1,600 = 10.0%**. Is that 2-point lift real?

- **H0:** variant = control (no difference). **H1:** variant ≠ control (two-tailed).
- Two-proportion z-test. Pooled rate = (128 + 160) / 3,200 = **0.09**.
- SE of the difference = √(0.09 × 0.91 × (1/1600 + 1/1600)) = √(0.0819 × 0.00125) ≈ **0.0101**.
- z = observed difference / SE = 0.02 / 0.0101 ≈ **1.98**.
- Two-tailed p-value ≈ **0.048**.

Since 0.048 < 0.05, you **just barely reject H0** — statistically significant. But notice how close it is: with a hair less data or a slightly smaller lift, you'd be on the other side of the line. That is exactly why you fix the sample size up front and do not "peek" and stop the moment p dips under 0.05 (peeking inflates your true Type I error well above 5%).

Contrast: if each group had only **n = 400**, the same 8% vs 10% gives z ≈ 0.99, p ≈ 0.32 — *not* significant. Same real effect, too little data to prove it. That is a **Type II error / low power**, not evidence the variant doesn't work.

## Interview angle

- **The #1 trap — define a p-value correctly:** it is *P(data this extreme | H0 true)*, NOT "the probability the null is true" and NOT "the probability the result was chance." Say it the right way and you're ahead of most candidates.
- **α vs p:** α is the threshold you choose *before*; p is what you *compute* from the data.
- **Type I vs Type II tradeoff:** lowering α cuts false positives but lowers power. Which error is worse depends on context (a false positive drug approval vs missing a real fraud signal).
- **Significant ≠ important:** with a huge n, a trivially small effect can be "significant." Always look at the **effect size / confidence interval**, not just p.
- **Power / sample size:** if a test is "not significant," ask whether it was **powered** to detect the effect before concluding "no difference."
- **Multiple comparisons & peeking** inflate Type I error — mention corrections (Bonferroni) or fixed sample sizes if asked about A/B pitfalls.

## Self-check
1. State what a p-value is, in one correct sentence.
2. You tighten α from 0.05 to 0.01. What happens to your Type I error rate, your Type II error rate, and your power?
3. A test with n = 50 per group shows p = 0.20 for what is genuinely a 2-point lift. What's most likely going on, and what should you do before concluding "no effect"?

## Answers
1. The probability of observing a result **at least as extreme as yours if the null hypothesis were true** (it says nothing directly about the probability that H0 itself is true).
2. **Type I error rate falls** (you demand stronger evidence, so fewer false positives), but **Type II error rises and power falls** — you'll miss more real effects. You traded false positives for false negatives.
3. Most likely **underpowered**: n = 50 is too small to detect a 2-point effect, so you fail to reject H0 — a **Type II error**, not proof the effect is absent. Do a **power / sample-size calculation** and collect more data (or run the test longer) before concluding there's no effect.
