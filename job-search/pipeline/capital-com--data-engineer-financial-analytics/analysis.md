2. **Pipeline architecture**: "How do you structure a multi-layer DWH?" → Staging (raw), Core (cleaned/typed), Data Mart (aggregated/business-ready).
3. **Debugging/root-cause**: "Tell me about a time you had to reverse-engineer an undocumented pipeline." → Use your PI Data Analytics work (inherited scripts, undocumented client schemas).
4. **Data quality**: "How do you ensure data quality in a production pipeline?" → Anomaly detection work, multi-agent QA workflow, validation checks at ingestion vs at mart.
5. **Autonomy**: "How do you handle ambiguous requirements?" → Procurement anomaly detection example: framed ambiguous spend concern into a hypothesis-driven investigation.
