**Project note — applying AI to accelerate analytical reporting:**

The project I'd highlight is an agentic client-report generation workflow I designed and built at PI Data Analytics. The problem was straightforward but time-consuming: after completing an analytics cycle for a client, someone had to manually assemble a structured progress report — pulling tables, writing summaries, formatting the narrative, and producing a client-ready document. This happened repeatedly across multiple clients and was entirely templated work that consumed hours per delivery cycle.

What I built was a multi-step agentic pipeline using Claude as the reasoning layer. The workflow ingested raw analytics outputs (processed data, KPI summaries, flagged anomalies), passed them through a sequence of specialised prompts that drafted each report section autonomously, and produced a structured, client-facing document — including an executive narrative, data tables, and recommendations — without manual assembly. I subsequently extended it to generate PowerPoint presentations from the same outputs.

What I learned: prompt design for structured document generation requires far more precision than I initially assumed. Early outputs were fluent but inconsistent in structure — sections would vary in length, tone, or level of detail across clients. The fix was to enforce explicit output schemas at each agent step rather than relying on free-form generation, and to add a validation pass that checked section completeness before assembly. I also learned that the highest-value part of the workflow was not the generation itself but the extraction layer upstream — getting the analytics outputs into a clean, structured format the agents could reason over reliably.

What I'd do differently: I'd invest earlier in building a reusable extraction schema rather than building prompt logic around ad hoc data shapes. I'd also introduce a human-review checkpoint for the executive narrative specifically, rather than treating the entire pipeline as hands-off — not because the output quality was poor, but because the narrative judgment involved in interpreting results for a client audience is the part where a human adds the most value and where errors are most costly.
